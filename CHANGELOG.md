# 更新日志

本项目遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/) 格式，
版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [Unreleased]

## [0.18.0] - 2026-07-14

### 变更

- Telegram 通知（流量预警/每日报告/延迟小节）排版从 `<pre>` 等宽块改为
  `<blockquote>` 引用块分组：消除 macOS 桌面客户端上的代码块底色、复制按钮
  与 CJK 等宽对齐错位问题，各端渲染观感一致；进度条字符 `█░` 换为 `▰▱`，
  延迟小节改为可折叠引用块并将采样时段并入小节标题。

## [0.17.3] - 2026-07-11

### 修复

- 默认时区从 `Asia/Shanghai` 改为 `Asia/Singapore`（`HELIOX_MON_TZ` 默认值），
  影响计费周期重置、每日报告整点对齐等；时区加载失败时的兜底固定偏移标签由
  易混淆的 `"CST"` 改为中性的 `"+08"`（偏移量本身不变，新加坡与上海同为 UTC+8
  无夏令时）。
- 前端延迟监控日期选择器 `formatDateValue` 改用浏览器本地日历日而非
  `toISOString()` 的 UTC 日历日：此前在 UTC+8 时区凌晨 0-8 点访问页面时，
  「今天」会被算成前一天，导致翻页/日期上限校验出错。

## [0.17.2] - 2026-07-09

### 修复

- 延迟趋势图放宽「无数据」判定阈值（从 1.5×粒度改为「一个粒度桶 + 90s 余量」）：
  采集节奏未对齐整分钟、偶尔某分钟落 0 个样本形成的单桶假空洞（ping 并未真丢），
  不再被误标为「无数据」；细粒度下需连续缺 ≥2 分钟才标灰，粗粒度下真实的多分钟
  中断仍照常显示。

## [0.17.1] - 2026-07-09

### 修复

- 延迟趋势图的「无数据」灰色区域左边缘往内缩了一格，未完整覆盖缺口起点；改为
  贴齐缺口前最后一个真实数据点，灰色区域两端与断线严丝合缝。

## [0.17.0] - 2026-07-09

### 新增

- 客户端延迟上报：新增 `POST /api/latency/report`（独立 `HELIOX_MON_REPORT_TOKEN`
  Bearer 认证，与登录密码隔离），客户端可上报到 VPS 的端到端 RTT。数据以
  `client:<name>` 命名空间写入现有 `latency_records`，完全复用降采样、保留策略与前端
  图表，效果等价于在 `PING_TARGETS` 无缝多一项。附参考客户端 `scripts/latency-client.sh`。
- 高精度测延端点 `GET /api/echo`：免认证、不写库、不打日志的 204 回显，供客户端在
  keep-alive 连接上测量端到端 TCP 净 RTT，排除服务端业务耗时干扰。

## [0.16.1] - 2026-07-09

### 修复

- CI/发布工作流的 setup-go 增加 `check-latest: true`，强制使用最新 Go 补丁版；
  用 go1.25.12 重建二进制，修复标准库 crypto/tls 漏洞 GO-2026-5856
  （0.16.0 由带漏洞的 go1.25.11 构建）。

## [0.16.0] - 2026-07-09

### 新增

- 延迟趋势图标注无数据缺口（Smokeping 风格）：监控停采（如服务器重启）时段折线
  断开并覆盖灰色「无数据」区域，不再用直线跨越缺口造成误导；tooltip 区分
  「无数据」（停采）与「丢包」（探测超时）。

### 修复

- 丢包「异常时长」统计不再把无数据缺口时段误计入（缺口前恰好是高丢包点时，
  原逻辑会把整段缺口算作异常持续时间）。

## [0.15.5] - 2026-07-04

### 变更

- 网络延迟小节改用「首行放实际采样时段」彻底解决 Telegram 复制按钮遮挡：首行
  `采样 HH:MM–HH:MM` 去替真实延迟数据「挡枪」，下方数据行不再被右上角复制按钮遮住，
  同时顺带展示昨日探测的真实时间覆盖。移除 0.15.4 的点号锚点。

## [0.15.4] - 2026-07-04

### 变更

- 网络延迟小节的行尾锚点由「空格 + 连字符」改为「空格 + 点号」，观感更和谐。

## [0.15.3] - 2026-07-04

### 变更

- 每日报告网络延迟小节恢复用 `<pre>` 等宽块（0.15.2 改的普通富文本行观感不佳）。
  改用「行尾锚点」解决复制按钮遮挡：给首行末尾追加一个连字符（非空格、不会被
  Telegram trim），正好落在按钮下被遮住，把真实数据顶出遮挡区。

## [0.15.2] - 2026-07-04

### 修复

- 每日报告网络延迟小节行尾（如「丢0%」）仍被 Telegram 代码块复制按钮遮挡：0.15.1
  的「行尾补空格」无效（客户端会 trim 行尾空格，且复制按钮定位在块右上角、遮挡的
  是首行）。改为该小节不再用 `<pre>` 等宽块，用加粗标签的普通富文本行（无复制按钮），
  数值以 `·` 分隔。

## [0.15.1] - 2026-07-04

### 修复

- 每日报告网络延迟小节的行尾（如「丢0%」）被 Telegram `<pre>` 代码块右上角复制
  按钮遮挡：行尾补留白撑宽代码块，让文字与按钮错开。

### 变更

- 网络延迟小节去掉丢包 >0 的 ⚠️ 图标（视觉更协调），仅保留「丢N%」文字。

## [0.15.0] - 2026-07-04

### 新增

- 通知弹层新增「发送日报预览」按钮（`POST /api/notify/daily-report`）：手动触发一次
  每日报告，便于在页面查看真实格式与内容。不影响自动调度（日报调度是纯内存定时器，
  本次发送不写「已发送」标记）；未配置 Telegram 时如实回显「未配置」。

## [0.14.0] - 2026-07-04

### 新增

- 每日流量报告新增**网络延迟**小节：按 `PING_TARGETS` 逐目标汇报昨日平均 RTT、
  最低 RTT 与丢包率（口径与「昨日用量」一致，昨日整天）；丢包 >0 标注 ⚠️，
  全丢包/无数据目标显示「无数据」。复用已有探测数据，无需新增配置。

### 变更

- Telegram 通知改用 HTML 富文本（`parse_mode=HTML`）：标题加粗，数据行放等宽
  `<pre>` 块对齐（修复此前纯文本在手机非等宽字体下对不齐的问题），收敛 emoji 为
  一套克制的语义符号并移除装饰性分隔线与标题图标。服务器名、探测目标名等用户可控
  字符串统一做 HTML 转义。日报 / 流量预警 / 测试三类消息一并改版。

## [0.13.0] - 2026-07-02

### 变更

- 每日流量报告全面改版，提升信息量与可读性：
  - 新增**环比**——昨日合计相对前日的涨跌百分比。
  - 新增**周期进度条**、**日均用量**与**月底预测**（照当前速度推算周期末总量，
    可能超限时标注 ⚠️）。
  - 统一单位为 GiB（原限额标签写 GB 但按 1024³ 计算，实为 GiB），日期附中文星期。
- 每日报告调度增加日志：启动/每轮打印「下次推送时间」（含时区），发送成功也记一条；
  开启开关但未配 Telegram 时启动即告警，便于排查「配了却不发送」。

## [0.12.1] - 2026-07-01

### 变更

- 通知 UI 改版：移除底部「通知」区块，改为顶栏服务器名旁的胶囊（仅在配置了 Telegram
  时显示，每日报告开启时显示推送时刻）。点击胶囊弹出面板查看配置状态并发送测试消息，
  点击外部 / Esc 关闭。

## [0.12.0] - 2026-07-01

### 新增

- 每日流量报告：可选定时通过 Telegram 推送昨日用量 + 本计费周期累计/剩余/重置日。
  由 `DAILY_REPORT_ENABLED` 开关、`DAILY_REPORT_HOUR` 指定推送时刻（按时区整点，默认 9 点），
  定时器对齐到下一次触发，进程重启不重复推送。
- 页面「通知」区块：显示 Telegram 是否已配置、每日报告开关/时刻，并提供「发送测试消息」
  按钮即时验证配置（成功/失败原因回显）。新增 `POST /api/notify/test`，`GET /api/config`
  补充 `daily_report` 字段。

## [0.11.0] - 2026-07-01

### 新增

- 工程化质量门禁：新增 GitHub Actions CI（lint / vet / test / 多平台构建 / govulncheck）。
- `golangci-lint` 配置（errcheck、staticcheck、govet、bodyclose、errorlint 等）。
- 核心逻辑单元测试：计费周期计算、配置与端口解析、虚拟网卡过滤、SQLite 迁移与流量查询。
- 新增 `LICENSE`（MIT）与本 `CHANGELOG`。
- `http.Server` 读取超时（`ReadHeaderTimeout` / `ReadTimeout` / `IdleTimeout`），
  防御 Slowloris 慢速连接；SSE 实时推送不设 `WriteTimeout` 以免长连接被掐断。

### 修复

- 显式处理此前被忽略的错误返回值（`rows.Scan` / `json.Encode` / `w.Write` /
  `db.Exec` / `server.Shutdown` 等），消除静默吞错。
- `make fmt` 在未安装 `goimports` 时报错：改用 `go run` 临时拉取；补全 Makefile `.PHONY` 目标。

### 变更

- 前端图表库（Chart.js / annotation 插件 / ECharts）由 CDN 改为本地 `go:embed` 内嵌，
  消除墙内 CDN 加载失败导致图表空白的隐患。
